## Content Development Workflows

### Workflow 1: From Fragmented Ideas to Complete Post
**When to use**: User provides scattered thoughts or bullet points

**Process**:
1. **Capture & Organize**: List all ideas and group related thoughts
2. **Find the Core**: Identify the central message or insight
3. **Choose Structure**: Select appropriate template based on content type
4. **Develop Narrative**: Expand ideas into coherent narrative flow
5. **Add Personality**: Inject personal voice and conversational elements
6. **Refine**: Ensure length, tone, and engagement elements are optimized

**Example Input**:
```
- AI is useful but people use it wrong
- Key is treating it as a partner not a tool
- Need conversational thinking
- Works much better for me
```

**Development Process**:
```
Core message: How to use AI effectively
Template: Observation â†’ Analysis
Hook: Personal discovery or conversation with friend
Structure: Set up problem â†’ Show different approaches â†’ Share insight
Personality: Conversational, relatable examples
```

### Workflow 2: From Topic to Fully Developed Post
**When to use**: User provides a topic/theme to expand

**Process**:
1. **Explore the Topic**: Ask clarifying questions if needed
   - What angle interests you most?
   - Any specific experience or example?
   - What should readers take away?
2. **Find Personal Connection**: Link topic to personal experience or observation
3. **Develop Hook**: Create compelling opening that relates to audience
4. **Build Content**:
   - Add 2-3 key points with supporting details
   - Include examples or anecdotes
   - Balance insight with accessibility
5. **Create Engagement**: End with question or call to action
6. **Generate Image**: Create visual that reinforces message

**Example Input**:
```
Want to write about "product thinking"
```

**Exploration Questions**:
- Which aspect of product thinking do you want to emphasize? (user perspective/data-driven/iterative thinking/business balance)
- Any specific examples or stories to share?
- What should readers take away after reading?

**Personal Discovery**:
- "I recently discovered..."
- "Last week, something made me rethink..."
- "After using X, I realized..."

**Surprising Statement**:
- "Unpopular opinion:..."
- "You might not believe this, but..."
- "Most people get this backwards..."

**Intriguing Observation**:
- "Noticed something interesting..."

- "What do you think?"

**Invitation to Share**:
- "Share your approach?"
- "Let's discuss in comments?"

## Image Prompt Patterns

### Concept-Based Images

**For AI/Technology Posts**:
```
Create a modern, professional illustration representing [CONCEPT]:
- Visual metaphor: [e.g., interconnected nodes for collaboration, lightbulb for innovation]
- Style: Clean, minimalist, tech-forward
- Color palette: Blues and teals with accent colors
- Mood: Innovative yet approachable
- Include subtle tech elements without being overly technical
```

**For Business/Entrepreneurship Posts**:
```
Create an engaging business concept illustration for [TOPIC]:
- Visual metaphor: [e.g., growth arrow, bridge for connection, puzzle for problem-solving]
- Style: Professional but warm and approachable
- Color palette: Vibrant yet professional (orange, blue, purple)
- Include human elements or relatable symbols
- Avoid generic stock photo feel
```

**For Product/Design Posts**:
```
Create a clean, design-focused illustration for [CONCEPT]:
- Visual elements: [e.g., interface elements, design tools, user journey]
- Style: Modern, minimal, emphasis on clarity
- Color palette: Consistent with tech/design aesthetics
- Show process or thinking, not just final product
- Balance simplicity with visual interest
```

### Abstract vs. Concrete

**Use Abstract Images When**:
- Discussing concepts, ideas, or principles
- Sharing insights or reflections
- Exploring philosophical or strategic topics

**Use Concrete Images When**:
- Showing specific tools or products
- Illustrating processes or workflows
- Sharing data or results
