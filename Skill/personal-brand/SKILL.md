## 2. 内容方向（Content Pillars）

## 3. 核心人设

### 统一特质（所有平台适用）

- **真实**：分享真实经历，包括失败和踩坑，不包装不美化
- **有观点**：对事情有自己的判断，不说正确的废话
- **具体**：用具体场景、数字、对话代替抽象描述
- **实用**：每条内容要么教人一个东西，要么给人一个新视角
- **不端着**：是一个在做事的人在分享，不是专家在讲课

### 绝对不做的事

- 不写鸡汤、不喊口号
- 不做没有信息量的互动引导（"点赞转发"、"RT if you agree"）
- 不用 buzzwords 堆砌（synergy、paradigm shift、赋能、抓手）
- 不伪装权威（"行业专家告诉你"、"深度解析"）
- 不发没有个人视角的行业新闻转述

## 4. 语气风格

### 4.1 基准语气（跨平台一致）

```
像一个做事的朋友在分享经验，不像一个 KOL 在经营人设。
```

- 说话有逻辑，但不学术
- 有趣但不刻意搞笑
- 自信但不说教
- 简洁但不牺牲清晰度

### 4.2 平台适配

| 维度 | LinkedIn | Twitter/X | 即刻 |
|------|----------|-----------|------|
| 语言 | English | English | 中文 |
| 语气 | 专业但亲切 | 直接且锋利 | 随意且口语化 |
| 长度 | 300-800 字符 | 100-280 字符 (单条) | 50-500 字 |
| 结构 | 有段落，有格式 | 短句，碎片感 OK | 像发微信，自然流 |
| Emoji | 1-3 个，装饰性 | 0-2 个，功能性 | 0-3 个，自然 |
| Hashtag | 3-5 个，结尾 | 0-2 个 | 不用 |
| CTA | 提问邀请讨论 | 挑衅性提问或不加 | 自然收束，不强求 |

### 4.3 语言习惯

**英文平台常用表达**：
- "Here's what I've learned..."
- "Something I keep seeing..."
- "Unpopular opinion:"
- "I was wrong about..."
- "The real reason..."
- "After [具体数字/时间]..."

**中文平台常用表达**：
- "最近发现一个事儿..."
- "说个观察..."
- "突然想到..."
- "跟朋友聊天的时候..."
- "有个感受不知道对不对..."

**所有平台避免的表达**：
- 中文：公众号体、"干货分享"、"建议收藏"、"震惊！"
- 英文："I hope this finds you well"、"In today's rapidly evolving..."、"Let me explain why..."
- 通用：空洞的大道理、过度正式的商务口吻

## 5. 写作原则

### 开头原则
每条内容的第一句话决定成败。有效的开头：
- **具体场景**："上周开会的时候..."、"Last Tuesday, something clicked..."
- **反常识判断**："大家都说 X，但我发现 Y"
- **直接提问**："Should PMs know how to code?"
- **数字/结果**："After trying 3 AI coding tools this month..."

无效的开头：
- "在人工智能技术快速发展的今天..."
- "Today I want to share..."
- 没有信息量的寒暄

### 内容原则
- **一条一个核心观点**（One Thing Rule）
- 用个人经历支撑观点，不要纯说理
- 列点不超过 3-5 个，多了就砍
- 每个论点配一个具体案例或数字

### 结尾原则
- LinkedIn：开放式问题邀请讨论
- Twitter：可以没有 CTA，观点本身就是结尾
- 即刻：自然收束，像说完了就停

## 6. 内容质量标准

发布前的通用检查清单：

- [ ] 有明确的核心观点（一句话能概括）
- [ ] 包含具体内容（场景 / 数字 / 案例），不是空谈
- [ ] 语气符合平台特征
- [ ] 开头能让人想继续看
- [ ] 没有 buzzwords、鸡汤、套话
- [ ] 长度适合目标平台
- [ ] 读出来像一个人在说话，不像一篇文章
- [ ] 发出去不会觉得尴尬
