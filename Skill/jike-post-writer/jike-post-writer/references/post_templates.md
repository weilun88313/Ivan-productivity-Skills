# 即刻帖子结构模板

## 模板 1：个人经历 → 感悟

**适合**：分享工作/生活中的发现和感受

```
[具体场景或事件]

[发生了什么 - 1-2句]

[我的感受/发现]

[（可选）延伸思考]
```

**示例**：
```
今天开会的时候，一个实习生问了句"这个功能真的有人用吗？"

全场安静了三秒。

后来我们去看了数据，发现这个花了两周做的功能，日活用户 23 个。

有时候最简单的问题，反而是最该问的问题。
```

## 模板 2：观察 → 分析

**适合**：分享行业观察、趋势判断

```
[一个有趣的观察]

[具体例子或对比]

[你的解读]
```

**示例**：
```
说个观察：身边做 AI 产品的朋友，做得好的都有一个共同点——他们不是在想"AI 能做什么"，而是在想"用户在哪里卡住了"。

技术视角 vs 用户视角，出发点不同，做出来的东西差别巨大。
```

## 模板 3：反直觉观点

**适合**：挑战常识、引发讨论

```
[一个反常识的判断]

[为什么这么说]

[具体依据或案例]

[（可选）你的结论]
```

**示例**：
```
越来越觉得，"快速迭代"可能是创业公司最大的陷阱之一。

不是说迭代不好，而是很多时候"快"变成了不思考的借口。上线 → 数据不好 → 改 → 还是不好 → 再改...

有时候慢下来想清楚一个核心问题，比连续迭代十个版本有用得多。
```

## 模板 4：实用分享

**适合**：分享工具、方法、技巧

```
[最近发现的好东西/方法]

[具体怎么用 - 简洁说明]

[效果/感受]
```

**示例**：
```
最近养成一个习惯：每天早上花 10 分钟，把今天最重要的一件事写下来。

不是 to-do list，就一件事。

坚持两周了，最大的变化是：下班的时候不再觉得"忙了一天但好像啥也没干"。
```

## 模板 5：碎片感悟（短帖）

**适合**：一句话或几句话的想法

**示例**：
```
产品经理最重要的能力可能不是"把事情做对"，而是"判断什么事情值得做"。
```

```
今天跟用户聊了一个小时，比看一周的数据报表收获大。
```

## 从碎片想法到帖子的方法

1. **收集**：列出所有想法碎片
2. **找核心**：这些碎片在说什么？一句话概括
3. **选结构**：根据核心观点选模板
4. **展开**：补充具体场景和细节
5. **精简**：删掉所有不必要的话，保持口语感

## 从话题到帖子的方法

1. **缩小范围**：大话题找小切口（"AI" → "用 AI 写代码的一个小发现"）
2. **找个人角度**：跟自己的经历挂钩
3. **具体化**：用具体的场景、数字、对话代替抽象描述
4. **说人话**：写完后读一遍，像不像在跟朋友说？

## 配图风格

仅在用户要求时生成配图：

- **风格**：清新、简约、有设计感
- **色调**：明亮温暖，避免太商务
- **类型**：概念插画、场景图、极简图形
- **避免**：stock photo 感、太多文字、过于抽象
